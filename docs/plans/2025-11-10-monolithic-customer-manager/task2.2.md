# Task 2.2: Implement Validation Service

**Agent Context:** You are creating a validation service that checks customer form data for required fields before API operations.

**Behavior:** Create a validationService object with a method to validate customer data and return validation errors.

**Acceptance Criteria:**
- Given the need for form data validation
- When calling validationService.validateCustomer(customer)
- Then it checks that name, email, and phone are not empty strings
- And it returns an array of ValidationError objects for any missing fields
- And it returns empty array if all required fields are present
- And validation messages are user-friendly (e.g., "Name is required")

**Test Scenarios:**
- **All fields valid:** Returns empty array
- **Name missing:** Returns ValidationError with field: 'name', message: 'Name is required'
- **Email missing:** Returns ValidationError with field: 'email', message: 'Email is required'
- **Phone missing:** Returns ValidationError with field: 'phone', message: 'Phone is required'
- **Multiple fields missing:** Returns array with multiple ValidationError objects
- **Empty strings:** Treats empty strings as missing (trim whitespace)
- **Undefined vs empty:** Handles both undefined and empty string values

**Agent-Specific Context:**
- **File to modify:** `src/components/CustomerManager.tsx` (add after storageService)
- **Tests to create:** None for this subtask (will be tested through integration)
- **Dependencies available:** Customer, ValidationError interfaces from Task 1.1
- **Interfaces to provide:** validationService object with method: validateCustomer(customer: Partial<Customer>): ValidationError[]

**Architecture Constraints:**
- Define validationService as a plain JavaScript object with methods
- Place after storageService, before component definition
- Check for both undefined and empty/whitespace-only strings
- Use trim() to handle whitespace
- Required fields: name, email, phone (id is generated by API)
- Simple presence validation only (no email format, phone format validation)
- Return array of ValidationError objects, not strings

**Definition of Done:**
- validationService object defined with validateCustomer method
- Method accepts Partial<Customer> to handle incomplete forms
- Returns ValidationError[] type
- All three required fields validated
- User-friendly error messages
- Code placed after storageService in file
